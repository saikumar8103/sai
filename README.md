Differentiable Time-Varying IIR Filtering for Real-Time Speech Denoising

1. Introduction
Speech denoising is a fundamental problem in audio signal processing, with applications in telecommunications, hearing aids, voice assistants, and real-time communication systems. Traditional denoising techniques often rely on fixed filters or non-adaptive methods that struggle in dynamic acoustic environments where noise characteristics change over time.
Infinite Impulse Response (IIR) filters are widely used due to their computational efficiency and ability to model recursive systems. However, classical IIR filters are typically static and not easily adaptable in real time. Moreover, their parameters are not inherently differentiable in a way that integrates seamlessly with modern machine learning frameworks.
This project introduces Differentiable Time-Varying IIR Filtering, a novel approach that combines:
	Classical signal processing (IIR filtering) 
	Deep learning (neural parameter estimation) 
	Differentiable programming (end-to-end training) 
The core idea is to design an IIR filter whose coefficients change over time and are generated by a neural network. By making the filtering process differentiable, the system can be trained using gradient-based optimization to minimize noise while preserving speech quality.

2. Methods
2.1 Problem Formulation
Given:
	Noisy speech signal: x[n]=s[n]+v[n]
	s[n]: clean speech 
	v[n]: noise 
Goal:
	Estimate clean speech s ̂[n]

2.2 Time-Varying IIR Filter
A standard IIR filter is defined as:
y[n]=∑_(k=0)^M▒b_k  x[n-k]-∑_(k=1)^N▒a_k  y[n-k]

In this project, coefficients become time-dependent:
y[n]=∑_(k=0)^M▒b_k [n]x[n-k]-∑_(k=1)^N▒a_k [n]y[n-k]

Where:
	b_k [n], a_k [n]: dynamically generated coefficients 

2.3 Neural Network for Coefficient Prediction
A neural network predicts filter coefficients at each time step:
Input Features:
	Short-time Fourier transform (STFT) 
	Log-magnitude spectrogram 
	Temporal context frames 
Model Architecture:
	1D CNN / RNN / LSTM / Transformer encoder 
	Fully connected layers 
	Output: filter coefficients a_k [n],b_k [n]

2.4 Differentiable Filtering
To enable training:
	The recursive IIR operation is implemented in a differentiable manner 
	Gradients are propagated through time (similar to RNNs) 
Techniques used:
	Backpropagation Through Time (BPTT) 
	Stability constraints on coefficients 
	Gradient clipping 

2.5 Stability Constraints
To ensure filter stability:
	Poles must lie inside the unit circle 
	Parameterization methods: 
	Reflection coefficients 
	Sigmoid/tanh mapping 
	Spectral normalization 

2.6 Loss Function
Common loss functions include:
	Mean Squared Error (MSE) 
L_MSE=∣∣s ̂[n]-s[n]∣∣^2

	Spectral Loss 
L_spec=∣∣STFT(s ̂)-STFT(s)∣∣^2

	Perceptual Loss 
	PESQ (Perceptual Evaluation of Speech Quality) 
	STOI (Short-Time Objective Intelligibility) 
Total Loss:
L=λ_1 L_MSE+λ_2 L_spec


2.7 Real-Time Processing Design
To enable real-time operation:
	Frame-by-frame processing 
	Low-latency buffering 
	Causal filtering (no future samples used)
	Pseudocode
Initialize neural network parameters θ
Initialize filter states y[-1], y[-2], ...

for each audio frame x[n]:
    
    # Step 1: Feature extraction
    features = ExtractFeatures(x[n])
    
    # Step 2: Predict filter coefficients
    (a[n], b[n]) = NeuralNetwork(features, θ)
    
    # Step 3: Apply time-varying IIR filter
    y[n] = 0
    
    for k = 0 to M:
        y[n] += b_k[n] * x[n-k]
    
    for k = 1 to N:
        y[n] -= a_k[n] * y[n-k]
    
    # Step 4: Store output
    output_signal.append(y[n])

# Training loop
for each batch:
    predicted = Model(noisy_input)
    loss = ComputeLoss(predicted, clean_signal)
    
    Backpropagate(loss)
    UpdateParameters(θ)

6. Limitations
Despite its advantages, the approach has several limitations:
6.1 Stability Issues
	Improper coefficient prediction can lead to unstable filters 
	Requires careful constraint design 
6.2 Training Complexity
	Backpropagation through recursive structures is computationally expensive 
	Gradient vanishing/exploding problems 

6.3 Limited Long-Term Context
	IIR filters rely on past outputs 
	May not capture very long dependencies as effectively as transformers 

6.4 Dataset Dependency
	Performance depends heavily on training data 
	Poor generalization to unseen noise types 

6.5 Implementation Challenges
	Real-time constraints require optimized implementation 
	Hardware deployment may need fixed-point approximation
